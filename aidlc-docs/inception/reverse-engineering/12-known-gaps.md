# 已知差距与改进方向

## P0 — 高优先级

### 持久化层
- **当前**: TaskLedger / OutcomeStore 为内存 dict，重启即丢
- **目标**: 接 SQLite/PostgreSQL，支持跨会话查询和审计
- **影响**: 治理链路无法跨会话追溯

### 决策环境快照完整性
- **当前**: DecisionRecord 记录了决策变量，但治理变量（routing trace、authority token）未关联落盘
- **目标**: 统一快照：决策变量 + 治理变量 + 环境变量一体化存储
- **影响**: 审计回放不完整

## P1 — 中优先级

### 场景插件化注册
- **当前**: 硬编码 `_SCENARIO_KEYWORDS` 字典，新场景必须改 engine.py
- **目标**: 插件化 ScenarioRegistry，支持外部注册场景
- **影响**: 扩展性受限

### 动态路由策略
- **当前**: 静态 YAML 规则，不感知运行时状态
- **目标**: 支持运行时指标（负载、成本、SLA）动态调整路由权重
- **影响**: 路由决策不够智能

### 多租户隔离
- **当前**: 单用户 session 级
- **目标**: tenant 级隔离（决策记忆、偏好、路由策略独立）
- **影响**: 无法支持多租户部署

## P2 — 低优先级

### 快照对比与回放 API
- **当前**: 快照数据已落盘，但缺少结构化查询和对比接口
- **目标**: 提供 diff API：同用户不同时间点的决策环境变化可量化

### Outcome 聚合分析
- **当前**: 单条记录查询
- **目标**: 跨场景、跨时间段的 outcome 聚合统计和趋势分析

### 语义搜索升级
- **当前**: DecisionMemory.recall_similar 使用轻量关键词匹配
- **目标**: 升级为 embedding 语义搜索

### biz/engine.py 拆分
- **当前**: 1300+ 行单文件，包含所有场景实现
- **目标**: 按场景拆分为独立模块
